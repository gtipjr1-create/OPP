# OPP Definition Of Done

- Behavior:
  Feature works for happy path and expected edge cases.
- Reliability:
  Core logic has unit/integration coverage for new branches.
- Accessibility:
  Keyboard operation, visible focus, and aria labels are present.
- UX states:
  Loading, error, and empty states are handled.
- Observability:
  New server/client failure points emit telemetry events.
- Quality gates:
  `npm run lint` and `npm test` pass before merge.
- Documentation:
  Update docs/checklists/templates when process or behavior changes.
